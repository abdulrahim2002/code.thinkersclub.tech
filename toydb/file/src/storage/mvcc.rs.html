      1 //! This module implements MVCC (Multi-Version Concurrency Control), a widely
      2 //! used method for ACID transactions and concurrency control. It allows
      3 //! multiple concurrent transactions to access and modify the same dataset,
      4 //! isolates them from each other, detects and handles conflicts, and commits
      5 //! their writes atomically as a single unit. It uses an underlying storage
      6 //! engine to store raw keys and values.
      7 //!
      8 //! VERSIONS
      9 //! ========
     10 //!
     11 //! MVCC handles concurrency control by managing multiple historical versions of
     12 //! keys, identified by a timestamp. Every write adds a new version at a higher
     13 //! timestamp, with deletes having a special tombstone value. For example, the
     14 //! keys a,b,c,d may have the following values at various logical timestamps (x
     15 //! is tombstone):
     16 //!
     17 //! Time
     18 //! 5
     19 //! 4  a4          
     20 //! 3      b3      x
     21 //! 2            
     22 //! 1  a1      c1  d1
     23 //!    a   b   c   d   Keys
     24 //!
     25 //! A transaction t2 that started at T=2 will see the values a=a1, c=c1, d=d1. A
     26 //! different transaction t5 running at T=5 will see a=a4, b=b3, c=c1.
     27 //!
     28 //! ToyDB uses logical timestamps with a sequence number stored in
     29 //! Key::NextVersion. Each new read-write transaction takes its timestamp from
     30 //! the current value of Key::NextVersion and then increments the value for the
     31 //! next transaction.
     32 //!
     33 //! ISOLATION
     34 //! =========
     35 //!
     36 //! MVCC provides an isolation level called snapshot isolation. Briefly,
     37 //! transactions see a consistent snapshot of the database state as of their
     38 //! start time. Writes made by concurrent or subsequent transactions are never
     39 //! visible to it. If two concurrent transactions write to the same key they
     40 //! will conflict and one of them must retry. A transaction's writes become
     41 //! atomically visible to subsequent transactions only when they commit, and are
     42 //! rolled back on failure. Read-only transactions never conflict with other
     43 //! transactions.
     44 //!
     45 //! Transactions write new versions at their timestamp, storing them as
     46 //! Key::Version(key, version) => value. If a transaction writes to a key and
     47 //! finds a newer version, it returns an error and the client must retry.
     48 //!
     49 //! Active (uncommitted) read-write transactions record their version in the
     50 //! active set, stored as Key::Active(version). When new transactions begin, they
     51 //! take a snapshot of this active set, and any key versions that belong to a
     52 //! transaction in the active set are considered invisible (to anyone except that
     53 //! transaction itself). Writes to keys that already have a past version in the
     54 //! active set will also return an error.
     55 //!
     56 //! To commit, a transaction simply deletes its record in the active set. This
     57 //! will immediately (and, crucially, atomically) make all of its writes visible
     58 //! to subsequent transactions, but not ongoing ones. If the transaction is
     59 //! cancelled and rolled back, it maintains a record of all keys it wrote as
     60 //! Key::TxnWrite(version, key), so that it can find the corresponding versions
     61 //! and delete them before removing itself from the active set.
     62 //!
     63 //! Consider the following example, where we have two ongoing transactions at
     64 //! time T=2 and T=5, with some writes that are not yet committed marked in
     65 //! parentheses.
     66 //!
     67 //! Active set: [2, 5]
     68 //!
     69 //! Time
     70 //! 5 (a5)
     71 //! 4  a4          
     72 //! 3      b3      x
     73 //! 2         (x)     (e2)
     74 //! 1  a1      c1  d1
     75 //!    a   b   c   d   e   Keys
     76 //!
     77 //! Here, t2 will see a=a1, d=d1, e=e2 (it sees its own writes). t5 will see
     78 //! a=a5, b=b3, c=c1. t2 does not see any newer versions, and t5 does not see
     79 //! the tombstone at c@2 nor the value e=e2, because version=2 is in its active
     80 //! set.
     81 //!
     82 //! If t2 tries to write b=b2, it receives an error and must retry, because a
     83 //! newer version exists. Similarly, if t5 tries to write e=e5, it receives an
     84 //! error and must retry, because the version e=e2 is in its active set.
     85 //!
     86 //! To commit, t2 can remove itself from the active set. A new transaction t6
     87 //! starting after the commit will then see c as deleted and e=e2. t5 will still
     88 //! not see any of t2's writes, because it's still in its local snapshot of the
     89 //! active set at the time it began.
     90 //!
     91 //! READ-ONLY AND TIME TRAVEL QUERIES
     92 //! =================================
     93 //!
     94 //! Since MVCC stores historical versions, it can trivially support time travel
     95 //! queries where a transaction reads at a past timestamp and has a consistent
     96 //! view of the database at that time.
     97 //!
     98 //! This is done by a transaction simply using a past version, as if it had
     99 //! started far in the past, ignoring newer versions like any other transaction.
    100 //! This transaction cannot write, as it does not have a unique timestamp (the
    101 //! original read-write transaction originally owned this timestamp).
    102 //!
    103 //! The only wrinkle is that the time-travel query must also know what the active
    104 //! set was at that version. Otherwise, it may see past transactions that committed
    105 //! after that time, which were not visible to the original transaction that wrote
    106 //! at that version. Similarly, if a time-travel query reads at a version that is
    107 //! still active, it should not see its in-progress writes, and after it commits
    108 //! a different time-travel query should not see those writes either, to maintain
    109 //! version consistency.
    110 //!
    111 //! To achieve this, every read-write transaction stores its active set snapshot
    112 //! in the storage engine as well, as Key::TxnActiveSnapshot, such that later
    113 //! time-travel queries can restore its original snapshot. Furthermore, a
    114 //! time-travel query can only see versions below the snapshot version, otherwise
    115 //! it could see spurious in-progress or since-committed versions.
    116 //!
    117 //! In the following example, a time-travel query at version=3 would see a=a1,
    118 //! c=c1, d=d1.
    119 //!
    120 //! Time
    121 //! 5
    122 //! 4  a4          
    123 //! 3      b3      x
    124 //! 2            
    125 //! 1  a1      c1  d1
    126 //!    a   b   c   d   Keys
    127 //!
    128 //! Read-only queries work similarly to time-travel queries, with one exception:
    129 //! they read at the next (current) version, i.e. Key::NextVersion, and use the
    130 //! current active set, storing the snapshot in memory only. Read-only queries
    131 //! do not increment the version sequence number in Key::NextVersion.
    132 //!
    133 //! GARBAGE COLLECTION
    134 //! ==================
    135 //!
    136 //! Normally, old versions would be garbage collected regularly, when they are
    137 //! no longer needed by active transactions or time-travel queries. However,
    138 //! ToyDB does not implement garbage collection, instead keeping all history
    139 //! forever, both out of laziness and also because it allows unlimited time
    140 //! travel queries (it's a feature, not a bug!).

    160 /// MVCC keys, using the KeyCode encoding which preserves the ordering and
    161 /// grouping of keys. Cow byte slices allow encoding borrowed values and
    162 /// decoding into owned values.
    163 #[derive(Debug, Deserialize, Serialize)]
    164 pub enum Key<'a> {
    165     /// The next available version.
    166     NextVersion,
    167     /// Active (uncommitted) transactions by version.
    168     TxnActive(Version),
    169     /// A snapshot of the active set at each version. Only written for
    170     /// versions where the active set is non-empty (excluding itself).
    171     TxnActiveSnapshot(Version),
    172     /// Keeps track of all keys written to by an active transaction (identified
    173     /// by its version), in case it needs to roll back.
    174     TxnWrite(
    175         Version,
    176         #[serde(with = "serde_bytes")]
    177         #[serde(borrow)]
    178         Cow<'a, [u8]>,
    179     ),
    180     /// A versioned key/value pair.
    181     Version(
    182         #[serde(with = "serde_bytes")]
    183         #[serde(borrow)]
    184         Cow<'a, [u8]>,
    185         Version,
    186     ),
    187     /// Unversioned non-transactional key/value pairs. These exist separately
    188     /// from versioned keys, i.e. the unversioned key "foo" is entirely
    189     /// independent of the versioned key "foo@7". These are mostly used
    190     /// for metadata.
    191     Unversioned(
    192         #[serde(with = "serde_bytes")]
    193         #[serde(borrow)]
    194         Cow<'a, [u8]>,
    195     ),
    196 }

    218 /// An MVCC-based transactional key-value engine. It wraps an underlying storage
    219 /// engine that's used for raw key/value storage.
    220 ///
    221 /// While it supports any number of concurrent transactions, individual read or
    222 /// write operations are executed sequentially, serialized via a mutex. There
    223 /// are two reasons for this: the storage engine itself is not thread-safe,
    224 /// requiring serialized access, and the Raft state machine that manages the
    225 /// MVCC engine applies commands one at a time from the Raft log, which will
    226 /// serialize them anyway.
    227 pub struct MVCC<E: Engine> {
    228     pub engine: Arc<Mutex<E>>,
    229 }

    267     /// Returns the status of the MVCC and storage engines.
    268     pub fn status(&self) -> Result<Status> {
    269         let mut engine = self.engine.lock()?;
    270         let versions = match engine.get(&Key::NextVersion.encode())? {
    271             Some(ref v) => Version::decode(v)? - 1,
    272             None => 0,
    273         };
    274         let active_txns = engine.scan_prefix(&KeyPrefix::TxnActive.encode()).count() as u64;
    275         Ok(Status { versions, active_txns, storage: engine.status()? })
    276     }

    300 /// A Transaction's state, which determines its write version and isolation. It
    301 /// is separate from Transaction to allow it to be passed around independently
    302 /// of the engine. There are two main motivations for this:
    303 ///
    304 /// - It can be exported via Transaction.state(), (de)serialized, and later used
    305 ///   to instantiate a new functionally equivalent Transaction via
    306 ///   Transaction::resume(). This allows passing the transaction between the
    307 ///   storage engine and SQL engine (potentially running on a different node)
    308 ///   across the Raft state machine boundary.
    309 ///
    310 /// - It can be borrowed independently of Engine, allowing references to it
    311 ///   in VisibleIterator, which would otherwise result in self-references.
    312 #[derive(Clone, Debug, PartialEq, Serialize, Deserialize)]
    313 pub struct TransactionState {
    314     /// The version this transaction is running at. Only one read-write
    315     /// transaction can run at a given version, since this identifies its
    316     /// writes.
    317     pub version: Version,
    318     /// If true, the transaction is read only.
    319     pub read_only: bool,
    320     /// The set of concurrent active (uncommitted) transactions, as of the start
    321     /// of this transaction. Their writes should be invisible to this
    322     /// transaction even if they're writing at a lower version, since they're
    323     /// not committed yet. Uses a BTreeSet for test determinism.
    324     pub active: BTreeSet<Version>,
    325 }

    327 impl encoding::Value for TransactionState {}

    329 impl TransactionState {
    330     /// Checks whether the given version is visible to this transaction.
    331     ///
    332     /// Future versions, and versions belonging to active transactions as of
    333     /// the start of this transaction, are never isible.
    334     ///
    335     /// Read-write transactions see their own writes at their version.
    336     ///
    337     /// Read-only queries only see versions below the transaction's version,
    338     /// excluding the version itself. This is to ensure time-travel queries see
    339     /// a consistent version both before and after any active transaction at
    340     /// that version commits its writes. See the module documentation for
    341     /// details.
    342     fn is_visible(&self, version: Version) -> bool {
    343         if self.active.get(&version).is_some() {
    344             false
    345         } else if self.read_only {
    346             version < self.version
    347         } else {
    348             version <= self.version
    349         }
    350     }
    351 }

    365 impl<E: Engine> Transaction<E> {
    366     /// Begins a new transaction in read-write mode. This will allocate a new
    367     /// version that the transaction can write at, add it to the active set, and
    368     /// record its active snapshot for time-travel queries.
    369     fn begin(engine: Arc<Mutex<E>>) -> Result<Self> {
    370         let mut session = engine.lock()?;
    371 
    372         // Allocate a new version to write at.
    373         let version = match session.get(&Key::NextVersion.encode())? {
    374             Some(ref v) => Version::decode(v)?,
    375             None => 1,
    376         };
    377         session.set(&Key::NextVersion.encode(), (version + 1).encode())?;
    378 
    379         // Fetch the current set of active transactions, persist it for
    380         // time-travel queries if non-empty, then add this txn to it.
    381         let active = Self::scan_active(&mut session)?;
    382         if !active.is_empty() {
    383             session.set(&Key::TxnActiveSnapshot(version).encode(), active.encode())?
    384         }
    385         session.set(&Key::TxnActive(version).encode(), vec![])?;
    386         drop(session);
    387 
    388         Ok(Self { engine, st: TransactionState { version, read_only: false, active } })
    389     }
    390 
    391     /// Begins a new read-only transaction. If version is given it will see the
    392     /// state as of the beginning of that version (ignoring writes at that
    393     /// version). In other words, it sees the same state as the read-write
    394     /// transaction at that version saw when it began.
    395     fn begin_read_only(engine: Arc<Mutex<E>>, as_of: Option<Version>) -> Result<Self> {
    396         let mut session = engine.lock()?;
    397 
    398         // Fetch the latest version.
    399         let mut version = match session.get(&Key::NextVersion.encode())? {
    400             Some(ref v) => Version::decode(v)?,
    401             None => 1,
    402         };
    403 
    404         // If requested, create the transaction as of a past version, restoring
    405         // the active snapshot as of the beginning of that version. Otherwise,
    406         // use the latest version and get the current, real-time snapshot.
    407         let mut active = BTreeSet::new();
    408         if let Some(as_of) = as_of {
    409             if as_of >= version {
    410                 return errinput!("version {as_of} does not exist");
    411             }
    412             version = as_of;
    413             if let Some(value) = session.get(&Key::TxnActiveSnapshot(version).encode())? {
    414                 active = BTreeSet::<Version>::decode(&value)?;
    415             }
    416         } else {
    417             active = Self::scan_active(&mut session)?;
    418         }
    419 
    420         drop(session);
    421 
    422         Ok(Self { engine, st: TransactionState { version, read_only: true, active } })
    423     }

    425     /// Resumes a transaction from the given state.
    426     fn resume(engine: Arc<Mutex<E>>, s: TransactionState) -> Result<Self> {
    427         // For read-write transactions, verify that the transaction is still
    428         // active before making further writes.
    429         if !s.read_only && engine.lock()?.get(&Key::TxnActive(s.version).encode())?.is_none() {
    430             return errinput!("no active transaction at version {}", s.version);
    431         }
    432         Ok(Self { engine, st: s })
    433     }

    435     /// Fetches the set of currently active transactions.
    436     fn scan_active(session: &mut MutexGuard<E>) -> Result<BTreeSet<Version>> {
    437         let mut active = BTreeSet::new();
    438         let mut scan = session.scan_prefix(&KeyPrefix::TxnActive.encode());
    439         while let Some((key, _)) = scan.next().transpose()? {
    440             match Key::decode(&key)? {
    441                 Key::TxnActive(version) => active.insert(version),
    442                 key => return errdata!("expected TxnActive key, got {key:?}"),
    443             };
    444         }
    445         Ok(active)
    446     }

    464     /// Commits the transaction, by removing it from the active set. This will
    465     /// immediately make its writes visible to subsequent transactions. Also
    466     /// removes its TxnWrite records, which are no longer needed.
    467     ///
    468     /// NB: commit does not flush writes to durable storage, since we rely on
    469     /// the Raft log for persistence.
    470     pub fn commit(self) -> Result<()> {
    471         if self.st.read_only {
    472             return Ok(());
    473         }
    474         let mut engine = self.engine.lock()?;
    475         let remove: Vec<_> = engine
    476             .scan_prefix(&KeyPrefix::TxnWrite(self.st.version).encode())
    477             .map_ok(|(k, _)| k)
    478             .try_collect()?;
    479         for key in remove {
    480             engine.delete(&key)?
    481         }
    482         engine.delete(&Key::TxnActive(self.st.version).encode())
    483     }
    484 
    485     /// Rolls back the transaction, by undoing all written versions and removing
    486     /// it from the active set. The active set snapshot is left behind, since
    487     /// this is needed for time travel queries at this version.
    488     pub fn rollback(self) -> Result<()> {
    489         if self.st.read_only {
    490             return Ok(());
    491         }
    492         let mut engine = self.engine.lock()?;
    493         let mut rollback = Vec::new();
    494         let mut scan = engine.scan_prefix(&KeyPrefix::TxnWrite(self.st.version).encode());
    495         while let Some((key, _)) = scan.next().transpose()? {
    496             match Key::decode(&key)? {
    497                 Key::TxnWrite(_, key) => {
    498                     rollback.push(Key::Version(key, self.st.version).encode()) // the version
    499                 }
    500                 key => return errdata!("expected TxnWrite, got {key:?}"),
    501             };
    502             rollback.push(key); // the TxnWrite record
    503         }
    504         drop(scan);
    505         for key in rollback.into_iter() {
    506             engine.delete(&key)?;
    507         }
    508         engine.delete(&Key::TxnActive(self.st.version).encode()) // remove from active set
    509     }

    521     /// Writes a new version for a key at the transaction's version. None writes
    522     /// a deletion tombstone. If a write conflict is found (either a newer or
    523     /// uncommitted version), a serialization error is returned.  Replacing our
    524     /// own uncommitted write is fine.
    525     fn write_version(&self, key: &[u8], value: Option<Vec<u8>>) -> Result<()> {
    526         if self.st.read_only {
    527             return Err(Error::ReadOnly);
    528         }
    529         let mut engine = self.engine.lock()?;
    530 
    531         // Check for write conflicts, i.e. if the latest key is invisible to us
    532         // (either a newer version, or an uncommitted version in our past). We
    533         // can only conflict with the latest key, since all transactions enforce
    534         // the same invariant.
    535         let from = Key::Version(
    536             key.into(),
    537             self.st.active.iter().min().copied().unwrap_or(self.st.version + 1),
    538         )
    539         .encode();
    540         let to = Key::Version(key.into(), u64::MAX).encode();
    541         if let Some((key, _)) = engine.scan(from..=to).last().transpose()? {
    542             match Key::decode(&key)? {
    543                 Key::Version(_, version) => {
    544                     if !self.st.is_visible(version) {
    545                         return Err(Error::Serialization);
    546                     }
    547                 }
    548                 key => return errdata!("expected Key::Version got {key:?}"),
    549             }
    550         }
    551 
    552         // Write the new version and its write record.
    553         //
    554         // NB: TxnWrite contains the provided user key, not the encoded engine
    555         // key, since we can construct the engine key using the version.
    556         engine.set(&Key::TxnWrite(self.st.version, key.into()).encode(), vec![])?;
    557         engine.set(&Key::Version(key.into(), self.st.version).encode(), bincode::serialize(&value))
    558     }

    560     /// Fetches a key's value, or None if it does not exist.
    561     pub fn get(&self, key: &[u8]) -> Result<Option<Vec<u8>>> {
    562         let mut engine = self.engine.lock()?;
    563         let from = Key::Version(key.into(), 0).encode();
    564         let to = Key::Version(key.into(), self.st.version).encode();
    565         let mut scan = engine.scan(from..=to).rev();
    566         while let Some((key, value)) = scan.next().transpose()? {
    567             match Key::decode(&key)? {
    568                 Key::Version(_, version) => {
    569                     if self.st.is_visible(version) {
    570                         return bincode::deserialize(&value);
    571                     }
    572                 }
    573                 key => return errdata!("expected Key::Version got {key:?}"),
    574             };
    575         }
    576         Ok(None)
    577     }
    578 
    579     /// Returns an iterator over the latest visible key/value pairs at the
    580     /// transaction's version.
    581     pub fn scan(&self, range: impl RangeBounds<Vec<u8>>) -> ScanIterator<E> {
    582         let start = match range.start_bound() {
    583             Bound::Excluded(k) => Bound::Excluded(Key::Version(k.into(), u64::MAX).encode()),
    584             Bound::Included(k) => Bound::Included(Key::Version(k.into(), 0).encode()),
    585             Bound::Unbounded => Bound::Included(Key::Version(vec![].into(), 0).encode()),
    586         };
    587         let end = match range.end_bound() {
    588             Bound::Excluded(k) => Bound::Excluded(Key::Version(k.into(), 0).encode()),
    589             Bound::Included(k) => Bound::Included(Key::Version(k.into(), u64::MAX).encode()),
    590             Bound::Unbounded => Bound::Excluded(KeyPrefix::Unversioned.encode()),
    591         };
    592         ScanIterator::new(self.engine.clone(), self.state().clone(), (start, end))
    593     }
    594 
    595     /// Scans keys under a given prefix.
    596     pub fn scan_prefix(&self, prefix: &[u8]) -> ScanIterator<E> {
    597         // Normally, KeyPrefix::Version will only match all versions of the
    598         // exact given key. We want all keys maching the prefix, so we chop off
    599         // the KeyCode byte slice terminator 0x0000 at the end.
    600         let mut prefix = KeyPrefix::Version(prefix.into()).encode();
    601         prefix.truncate(prefix.len() - 2);
    602         let range = keycode::prefix_range(&prefix);
    603         ScanIterator::new(self.engine.clone(), self.state().clone(), range)
    604     }
    605 }

    607 /// An iterator over the latest live and visible key/value pairs for the txn.
    608 ///
    609 /// The (single-threaded) engine is protected by a mutex, and holding the mutex
    610 /// for the duration of the iteration can cause deadlocks (e.g. when the local
    611 /// SQL engine pulls from two tables concurrently during a join). Instead, we
    612 /// pull and buffer a batch of rows at a time, and release the mutex in between.
    613 ///
    614 /// This does not implement DoubleEndedIterator (reverse scans), since the SQL
    615 /// layer doesn't currently need it.
    616 #[allow(clippy::type_complexity)]
    617 pub struct ScanIterator<E: Engine> {
    618     /// The engine.
    619     engine: Arc<Mutex<E>>,
    620     /// The transaction state.
    621     txn: TransactionState,
    622     /// A buffer of live and visible key/value pairs to emit.
    623     buffer: VecDeque<(Vec<u8>, Vec<u8>)>,
    624     /// The remaining range after the buffer.
    625     remainder: Option<(Bound<Vec<u8>>, Bound<Vec<u8>>)>,
    626 }
    627 
    628 /// Implement Clone manually. Deriving it requires Engine: Clone.
    629 impl<E: Engine> Clone for ScanIterator<E> {
    630     fn clone(&self) -> Self {
    631         Self {
    632             engine: self.engine.clone(),
    633             txn: self.txn.clone(),
    634             buffer: self.buffer.clone(),
    635             remainder: self.remainder.clone(),
    636         }
    637     }
    638 }
    639 
    640 impl<E: Engine> ScanIterator<E> {
    641     /// The number of live keys to pull from the engine at a time.
    642     #[cfg(not(test))]
    643     const BUFFER_SIZE: usize = 1000;
    644     /// Pull only 2 keys in tests, to exercise this more often.
    645     #[cfg(test)]
    646     const BUFFER_SIZE: usize = 2;
    647 
    648     /// Creates a new scan iterator.
    649     fn new(
    650         engine: Arc<Mutex<E>>,
    651         txn: TransactionState,
    652         range: (Bound<Vec<u8>>, Bound<Vec<u8>>),
    653     ) -> Self {
    654         let buffer = VecDeque::with_capacity(Self::BUFFER_SIZE);
    655         Self { engine, txn, buffer, remainder: Some(range) }
    656     }
    657 
    658     /// Fills the buffer, if there's any pending items.
    659     fn fill_buffer(&mut self) -> Result<()> {
    660         // Check if there's anything to buffer.
    661         if self.buffer.len() >= Self::BUFFER_SIZE {
    662             return Ok(());
    663         }
    664         let Some(range) = self.remainder.take() else { return Ok(()) };
    665         let range_end = range.1.clone();
    666 
    667         let mut engine = self.engine.lock()?;
    668         let mut iter = VersionIterator::new(&self.txn, engine.scan(range)).peekable();
    669         while let Some((key, _, value)) = iter.next().transpose()? {
    670             // If the next key equals this one, we're not at the latest version.
    671             match iter.peek() {
    672                 Some(Ok((next, _, _))) if next == &key => continue,
    673                 Some(Err(err)) => return Err(err.clone()),
    674                 Some(Ok(_)) | None => {}
    675             }
    676             // Decode the value, and skip deleted keys (tombstones).
    677             let Some(value) = bincode::deserialize(&value)? else { continue };
    678             self.buffer.push_back((key, value));
    679             // If we filled the buffer, save the remaining range (if any) and
    680             // return. peek() has already buffered next(), so pull it.
    681             if self.buffer.len() == Self::BUFFER_SIZE {
    682                 if let Some((next, version, _)) = iter.next().transpose()? {
    683                     // We have to re-encode it as a raw engine key, since we
    684                     // only have access to the decoded MVCC user key.
    685                     let range_start = Bound::Included(Key::Version(next.into(), version).encode());
    686                     self.remainder = Some((range_start, range_end));
    687                 }
    688                 return Ok(());
    689             }
    690         }
    691         Ok(())
    692     }
    693 }
    694 
    695 impl<E: Engine> Iterator for ScanIterator<E> {
    696     type Item = Result<(Vec<u8>, Vec<u8>)>;
    697     fn next(&mut self) -> Option<Self::Item> {
    698         if self.buffer.is_empty() {
    699             if let Err(error) = self.fill_buffer() {
    700                 return Some(Err(error));
    701             }
    702         }
    703         self.buffer.pop_front().map(Ok)
    704     }
    705 }

    707 /// An iterator that decodes raw engine key/value pairs into MVCC key/value
    708 /// versions, and skips invisible versions. Helper for ScanIterator.
    709 struct VersionIterator<'a, I: engine::ScanIterator> {
    710     /// The transaction the scan is running in.
    711     txn: &'a TransactionState,
    712     /// The inner engine scan iterator.
    713     inner: I,
    714 }
    715 
    716 impl<'a, I: engine::ScanIterator> VersionIterator<'a, I> {
    717     /// Creates a new MVCC version iterator for the given engine iterator.
    718     fn new(txn: &'a TransactionState, inner: I) -> Self {
    719         Self { txn, inner }
    720     }
    721 
    722     // Fallible next(). Returns the next visible key/version/value tuple.
    723     #[allow(clippy::type_complexity)]
    724     fn try_next(&mut self) -> Result<Option<(Vec<u8>, Version, Vec<u8>)>> {
    725         while let Some((key, value)) = self.inner.next().transpose()? {
    726             let Key::Version(key, version) = Key::decode(&key)? else {
    727                 return errdata!("expected Key::Version got {key:?}");
    728             };
    729             if !self.txn.is_visible(version) {
    730                 continue;
    731             }
    732             return Ok(Some((key.into_owned(), version, value)));
    733         }
    734         Ok(None)
    735     }
    736 }
    737 
    738 impl<'a, I: engine::ScanIterator> Iterator for VersionIterator<'a, I> {
    739     type Item = Result<(Vec<u8>, Version, Vec<u8>)>;
    740     fn next(&mut self) -> Option<Self::Item> {
    741         self.try_next().transpose()
    742     }
    743 }

    827     impl goldenscript::Runner for MVCCRunner {
    828         fn run(&mut self, command: &goldenscript::Command) -> Result<String, Box<dyn Error>> {
    829             let mut output = String::new();
    830             let mut tags = command.tags.clone();
    831 
    832             match command.name.as_str() {
    833                 // txn: begin [readonly] [as_of=VERSION]
    834                 "begin" => {
    835                     let name = Self::txn_name(&command.prefix)?;
    836                     if self.txns.contains_key(name) {
    837                         return Err(format!("txn {name} already exists").into());
    838                     }
    839                     let mut args = command.consume_args();
    840                     let readonly = match args.next_pos().map(|a| a.value.as_str()) {
    841                         Some("readonly") => true,
    842                         None => false,
    843                         Some(v) => return Err(format!("invalid argument {v}").into()),
    844                     };
    845                     let as_of = args.lookup_parse("as_of")?;
    846                     args.reject_rest()?;
    847                     let txn = match (readonly, as_of) {
    848                         (false, None) => self.mvcc.begin()?,
    849                         (true, None) => self.mvcc.begin_read_only()?,
    850                         (true, Some(v)) => self.mvcc.begin_as_of(v)?,
    851                         (false, Some(_)) => return Err("as_of only valid for read-only txn".into()),
    852                     };
    853                     self.txns.insert(name.to_string(), txn);
    854                 }
    855 
    856                 // txn: commit
    857                 "commit" => {
    858                     let name = Self::txn_name(&command.prefix)?;
    859                     let txn = self.txns.remove(name).ok_or(format!("unknown txn {name}"))?;
    860                     command.consume_args().reject_rest()?;
    861                     txn.commit()?;
    862                 }
    863 
    864                 // txn: delete KEY...
    865                 "delete" => {
    866                     let txn = self.get_txn(&command.prefix)?;
    867                     let mut args = command.consume_args();
    868                     for arg in args.rest_pos() {
    869                         let key = decode_binary(&arg.value);
    870                         txn.delete(&key)?;
    871                     }
    872                     args.reject_rest()?;
    873                 }
    874 
    875                 // dump
    876                 "dump" => {
    877                     command.consume_args().reject_rest()?;
    878                     let mut engine = self.mvcc.engine.lock().unwrap();
    879                     let mut scan = engine.scan(..);
    880                     while let Some((key, value)) = scan.next().transpose()? {
    881                         let fmtkv = format::MVCC::<format::Raw>::key_value(&key, &value);
    882                         let rawkv = format::Raw::key_value(&key, &value);
    883                         writeln!(output, "{fmtkv} [{rawkv}]")?;
    884                     }
    885                 }
    886 
    887                 // txn: get KEY...
    888                 "get" => {
    889                     let txn = self.get_txn(&command.prefix)?;
    890                     let mut args = command.consume_args();
    891                     for arg in args.rest_pos() {
    892                         let key = decode_binary(&arg.value);
    893                         let value = txn.get(&key)?;
    894                         let fmtkv = format::Raw::key_maybe_value(&key, value.as_deref());
    895                         writeln!(output, "{fmtkv}")?;
    896                     }
    897                     args.reject_rest()?;
    898                 }
    899 
    900                 // get_unversioned KEY...
    901                 "get_unversioned" => {
    902                     Self::no_txn(command)?;
    903                     let mut args = command.consume_args();
    904                     for arg in args.rest_pos() {
    905                         let key = decode_binary(&arg.value);
    906                         let value = self.mvcc.get_unversioned(&key)?;
    907                         let fmtkv = format::Raw::key_maybe_value(&key, value.as_deref());
    908                         writeln!(output, "{fmtkv}")?;
    909                     }
    910                     args.reject_rest()?;
    911                 }
    912 
    913                 // import [VERSION] KEY=VALUE...
    914                 "import" => {
    915                     Self::no_txn(command)?;
    916                     let mut args = command.consume_args();
    917                     let version = args.next_pos().map(|a| a.parse()).transpose()?;
    918                     let mut txn = self.mvcc.begin()?;
    919                     if let Some(version) = version {
    920                         if txn.version() > version {
    921                             return Err(format!("version {version} already used").into());
    922                         }
    923                         while txn.version() < version {
    924                             txn = self.mvcc.begin()?;
    925                         }
    926                     }
    927                     for kv in args.rest_key() {
    928                         let key = decode_binary(kv.key.as_ref().unwrap());
    929                         let value = decode_binary(&kv.value);
    930                         if value.is_empty() {
    931                             txn.delete(&key)?;
    932                         } else {
    933                             txn.set(&key, value)?;
    934                         }
    935                     }
    936                     args.reject_rest()?;
    937                     txn.commit()?;
    938                 }
    939 
    940                 // txn: resume JSON
    941                 "resume" => {
    942                     let name = Self::txn_name(&command.prefix)?;
    943                     let mut args = command.consume_args();
    944                     let raw = &args.next_pos().ok_or("state not given")?.value;
    945                     args.reject_rest()?;
    946                     let state: TransactionState = serde_json::from_str(raw)?;
    947                     let txn = self.mvcc.resume(state)?;
    948                     self.txns.insert(name.to_string(), txn);
    949                 }
    950 
    951                 // txn: rollback
    952                 "rollback" => {
    953                     let name = Self::txn_name(&command.prefix)?;
    954                     let txn = self.txns.remove(name).ok_or(format!("unknown txn {name}"))?;
    955                     command.consume_args().reject_rest()?;
    956                     txn.rollback()?;
    957                 }
    958 
    959                 // txn: scan [RANGE]
    960                 "scan" => {
    961                     let txn = self.get_txn(&command.prefix)?;
    962                     let mut args = command.consume_args();
    963                     let range =
    964                         parse_key_range(args.next_pos().map(|a| a.value.as_str()).unwrap_or(".."))?;
    965                     args.reject_rest()?;
    966 
    967                     let kvs: Vec<_> = txn.scan(range).try_collect()?;
    968                     for (key, value) in kvs {
    969                         writeln!(output, "{}", format::Raw::key_value(&key, &value))?;
    970                     }
    971                 }
    972 
    973                 // txn: scan_prefix PREFIX
    974                 "scan_prefix" => {
    975                     let txn = self.get_txn(&command.prefix)?;
    976                     let mut args = command.consume_args();
    977                     let prefix = decode_binary(&args.next_pos().ok_or("prefix not given")?.value);
    978                     args.reject_rest()?;
    979 
    980                     let kvs: Vec<_> = txn.scan_prefix(&prefix).try_collect()?;
    981                     for (key, value) in kvs {
    982                         writeln!(output, "{}", format::Raw::key_value(&key, &value))?;
    983                     }
    984                 }
    985 
    986                 // txn: set KEY=VALUE...
    987                 "set" => {
    988                     let txn = self.get_txn(&command.prefix)?;
    989                     let mut args = command.consume_args();
    990                     for kv in args.rest_key() {
    991                         let key = decode_binary(kv.key.as_ref().unwrap());
    992                         let value = decode_binary(&kv.value);
    993                         txn.set(&key, value)?;
    994                     }
    995                     args.reject_rest()?;
    996                 }
    997 
    998                 // set_unversioned KEY=VALUE...
    999                 "set_unversioned" => {
   1000                     Self::no_txn(command)?;
   1001                     let mut args = command.consume_args();
   1002                     for kv in args.rest_key() {
   1003                         let key = decode_binary(kv.key.as_ref().unwrap());
   1004                         let value = decode_binary(&kv.value);
   1005                         self.mvcc.set_unversioned(&key, value)?;
   1006                     }
   1007                     args.reject_rest()?;
   1008                 }
   1009 
   1010                 // txn: state
   1011                 "state" => {
   1012                     command.consume_args().reject_rest()?;
   1013                     let txn = self.get_txn(&command.prefix)?;
   1014                     let state = txn.state();
   1015                     write!(
   1016                         output,
   1017                         "v{} {} active={{{}}}",
   1018                         state.version,
   1019                         if state.read_only { "ro" } else { "rw" },
   1020                         state.active.iter().sorted().join(",")
   1021                     )?;
   1022                 }
   1023 
   1024                 // status
   1025                 "status" => writeln!(output, "{:#?}", self.mvcc.status()?)?,
   1026 
   1027                 name => return Err(format!("invalid command {name}").into()),
   1028             }
   1029 
   1030             // If requested, output engine operations.
   1031             if tags.remove("ops") {
   1032                 while let Ok(op) = self.op_rx.try_recv() {
   1033                     match op {
   1034                         Operation::Delete { key } => {
   1035                             let fmtkey = format::MVCC::<format::Raw>::key(&key);
   1036                             let rawkey = format::Raw::key(&key);
   1037                             writeln!(output, "engine delete {fmtkey} [{rawkey}]")?
   1038                         }
   1039                         Operation::Flush => writeln!(output, "engine flush")?,
   1040                         Operation::Set { key, value } => {
   1041                             let fmtkv = format::MVCC::<format::Raw>::key_value(&key, &value);
   1042                             let rawkv = format::Raw::key_value(&key, &value);
   1043                             writeln!(output, "engine set {fmtkv} [{rawkv}]")?
   1044                         }
   1045                     }
   1046                 }
   1047             }
   1048 
   1049             if let Some(tag) = tags.iter().next() {
   1050                 return Err(format!("unknown tag {tag}").into());
   1051             }
   1052 
   1053             Ok(output)
   1054         }
   1055 
   1056         // Drain unhandled engine operations.
   1057         fn end_command(&mut self, _: &goldenscript::Command) -> Result<String, Box<dyn Error>> {
   1058             while self.op_rx.try_recv().is_ok() {}
   1059             Ok(String::new())
   1060         }
   1061     }

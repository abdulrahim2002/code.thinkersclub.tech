      3 ## Data Types
      4 
      5 The following data types are supported:
      6 
      7 * `BOOLEAN` (`BOOL`): logical truth values, i.e. true and false.
      8 * `FLOAT` (`DOUBLE`): 64-bit signed floating point numbers, using [IEEE 754 `binary64`](https://en.wikipedia.org/wiki/binary64) encoding. Supports magnitudes of 10‚Åª¬≥‚Å∞‚Å∑ to 10¬≥‚Å∞‚Å∏ with 53-bit precision (~15 significant figures), as well as the special values infinity and NaN.
      9 * `INTEGER` (`INT`): 64-bit signed integer numbers with a range of ¬±2‚Å∂¬≥-1.
     10 * `STRING` (`TEXT`, `VARCHAR`): UTF-8 encoded strings.
     11 
     12 In addition, the special `NULL` value is used for an unknown value, following the rules of [three-valued logic](https://en.wikipedia.org/wiki/Three-valued_logic).
     13 
     14 Numeric types are not interchangable; a float value (even without a fractional part) cannot be stored in an integer column and vice-versa.

     16 ## SQL Syntax
     17 
     18 ### Keywords
     19 
     20 Keywords are reserved words with special meaning in SQL statements. They are case-insensitive, and must be quoted with `"` to be used as identifiers. The complete list is:
     21 
     22 `AS`, `ASC`, `AND`, `BEGIN`, `BOOL`, `BOOLEAN`, `BY`, `COMMIT`, `CREATE`, `CROSS`, `DEFAULT`,`DELETE`, `DESC`, `DOUBLE`, `DROP`, `EXISTS`, `EXPLAIN`, `FALSE`, `FLOAT`, `FROM`, `GROUP`, `HAVING`, `IF`, `INDEX`, `INFINITY`, `INNER`, `INSERT`, `INT`, `INTEGER`, `INTO`, `IS`, `JOIN`, `KEY`, `LEFT`, `LIKE`, `LIMIT`, `NAN`, `NOT`, `NULL`, `OF`, `OFFSET`, `ON`, `ONLY`, `OR`, `ORDER`, `OUTER`, `PRIMARY`, `READ`, `REFERENCES`, `RIGHT`, `ROLLBACK`, `SELECT`, `SET`, `STRING`, `SYSTEM`, `TABLE`, `TEXT`, `TIME`, `TRANSACTION`, `TRUE`, `UNIQUE`, `UPDATE`, `VALUES`, `VARCHAR`, `WHERE`, `WRITE`
     23 
     24 ### Identifiers
     25 
     26 Identifiers are names for database objects such as tables and columns. Unless quoted with `"`, they must begin with a Unicode letter followed by any combination of letters, numbers, and `_`, and cannot be reserved keywords. `""` can be used to escape a double quote character. They are always converted to lowercase.
     27 
     28 ### Constants
     29 
     30 #### Named constants
     31 
     32 The following keywords evaluate to constants:
     33 
     34 * `FALSE`: the boolean false value.
     35 * `INFINITY`: the floating-point value for infinity.
     36 * `NAN`: the floating-point value for NaN (not a number).
     37 * `NULL`: an unknown value.
     38 * `TRUE`: the boolean true value.
     39 
     40 #### String literals
     41 
     42 String literals are surrounded by single quotes `'`, and can contain any valid UTF-8 character. Single quotes must be escaped by an additional single quote, i.e. `''`, no other escape sequences are supported. For example:
     43 
     44 ```
     45 'A string with ''quotes'' and emojis üòÄ'
     46 ```
     47 
     48 #### Numeric literals
     49 
     50 Sequences of digits `0-9` are parsed as a 64-bit signed integer. Numbers with decimal points or in scientific notation are parsed as 64-bit floating point numbers. The following pattern is supported:
     51 
     52 ```
     53 999[.[999]][e[+-]999]
     54 ```
     55 
     56 The `-` prefix operator can be used to take negative numbers.
     57 
     58 ### Expressions
     59 
     60 Expressions can be used wherever a value is expected, e.g. as `SELECT` columns nd `INSERT` values. They are made up of constants, a column references, an operator invocations, and a function calls.
     61 
     62 Column references can either be unqualified, e.g. `name`, or prefixed with the relation identifier separated by `.`, e.g. `person.name`. Unqualified identifiers must be unambiguous.
     63 
     64 ## SQL Operators
     65 
     66 ### Logical operators
     67 
     68 Logical operators apply standard logic operations on boolean operands.
     69 
     70 * `AND`: the logical conjunction, e.g. `TRUE AND TRUE` yields `TRUE`.
     71 * `OR`: the logical disjunction, e.g. `TRUE OR FALSE` yields `TRUE`.
     72 * `NOT`: the logical negation, e.g. `NOT TRUE` yields `FALSE`.
     73 
     74 The complete truth tables are:
     75 
     76 | `AND`       | `TRUE`  | `FALSE` | `NULL`  |
     77 |-------------|---------|---------|---------|
     78 | **`TRUE`**  | `TRUE`  | `FALSE` | `NULL`  |
     79 | **`FALSE`** | `FALSE` | `FALSE` | `FALSE` |
     80 | **`NULL`**  | `NULL`  | `FALSE` | `NULL`  |
     81 
     82 | `OR`        | `TRUE` | `FALSE` | `NULL` |
     83 |-------------|--------|---------|--------|
     84 | **`TRUE`**  | `TRUE` | `TRUE`  | `TRUE` |
     85 | **`FALSE`** | `TRUE` | `FALSE` | `NULL` |
     86 | **`NULL`**  | `TRUE` | `NULL`  | `NULL` |
     87 
     88 | `NOT`       |         |
     89 |-------------|---------|
     90 | **`TRUE`**  | `FALSE` |
     91 | **`FALSE`** | `TRUE`  |
     92 | **`NULL`**  | `NULL`  |
     93 
     94 ### Comparison operators
     95 
     96 Comparison operators compare values of the same data type, and return `TRUE` if the comparison holds or `FALSE` otherwise. `INTEGER` and `FLOAT` values are interchangeable. `STRING` comparisons use the string's byte values, i.e. case-sensitive with `'B' < 'a'` due to their UTF-8 code points. `FALSE` is considered lesser than `TRUE`. Comparison with `NULL` always yields `NULL` (even `NULL = NULL`).
     97 
     98 Binary operators:
     99 
    100 * `=`: equality, e.g. `1 = 1` yields `TRUE`.
    101 * `!=`: inequality, e.g. `1 != 2` yields `TRUE`.
    102 * `>`: greater than, e.g. `2 > 1` yields `TRUE`.
    103 * `>=`: greater than or equal, e.g. `1 >= 1` yields `TRUE`.
    104 * `<`: lesser than, e.g. `1 < 2` yields `TRUE`.
    105 * `<=`: lesser than or equal, e.g. `1 <= 1` yields `TRUE`.
    106 
    107 Unary operators:
    108 
    109 * `IS NULL`: checks if the value is `NULL`, e.g. `NULL IS NULL` yields `TRUE`.
    110 * `IS NOT NULL`: checks if the value is not `NULL`, e.g. `TRUE IS NOT NULL` yields `TRUE`.
    111 * `IS NAN`: checks if the value is a float `NAN`, e.g. `NAN IS NAN` yields `TRUE`. Errors on 
    112   non-float datatypes, except `NULL` which yields `NULL`.
    113 * `IS NOT NAN`: checks if the value is not a float `NAN`, e.g. `3.14 IS NOT NAN` yields `TRUE`.
    114 
    115 ### Mathematical operators
    116 
    117 Mathematical operators apply standard math operations on numeric (`INTEGER` or `FLOAT`) operands. If either operand is a `FLOAT`, both operands are converted to `FLOAT` and the result is a `FLOAT`. If either operand is `NULL`, the result is `NULL`. The special values `INFINITY` and `NAN` are handled according to the IEEE 754 spec.
    118 
    119 For `INTEGER` operands, failure conditions such as overflow and division by zero yield an error. For `FLOAT` operands, these return `INFINITY` or `NAN` as appropriate.
    120 
    121 Binary operators:
    122 
    123 * `+`: addition, e.g. `1 + 2` yields `3`.
    124 * `-`: subtraction, e.g. `3 - 2` yields `1`.
    125 * `*`: multiplication, e.g. `3 * 2` yields `6`.
    126 * `/`: division, e.g. `6 / 2` yields `3`.
    127 * `^`: exponentiation, e.g. `2 ^ 4` yields `16`.
    128 * `%`: remainder, e.g. `8 % 3` yields `2`. Unlike modulo, the result has the sign of the dividend.
    129 
    130 Unary operators:
    131 
    132 * `+` (prefix): identity, e.g. `+1` yields `1`.
    133 * `-` (prefix): negation, e.g. `- -2` yields `2`.
    134 * `!` (postfix): factorial, e.g. `5!` yields `15`.
    135 
    136 ### String operators
    137 
    138 String operators operate on string operands.
    139 
    140 * `LIKE`: compares a string with the given pattern, using `%` as multi-character wildcard and `_` as single-character wildcard, returning `TRUE` if the string matches the pattern - e.g. `'abc' LIKE 'a%'` yields `TRUE`.
    141 
    142 ### Operator precedence
    143 
    144 The operator precedence (order of operations) is as follows:
    145 
    146 | Precedence | Operator                | Associativity |
    147 |------------|-------------------------|---------------|
    148 | 10         | `+`, `-` (prefix)       | Right         |
    149 | 9          | `!` (postfix)           | Left          |
    150 | 8          | `^`                     | Right         |
    151 | 7          | `*`, `/`, `%`           | Left          |
    152 | 6          | `+`, `-`                | Left          |
    153 | 5          | `>`, `>=`, `<`, `<=`    | Left          |
    154 | 4          | `=`, `!=`, `LIKE`, `IS` | Left          |
    155 | 3          | `NOT`                   | Right         |
    156 | 2          | `AND`                   | Left          |
    157 | 1          | `OR`                    | Left          |
    158 
    159 Precedence can be overridden by wrapping an expression in parentheses, e.g. `(1 + 2) * 3`.
    160 
    161 ### Functions
    162 
    163 * `sqrt(expr)`: returns the square root of a numerical argument.
    164 
    165 ### Aggregate functions
    166 
    167 Aggregate function aggregate an expression across all rows, optionally grouped into buckets given by `GROUP BY`, and results can be filtered via `HAVING`.
    168 
    169 * `AVG(expr)`: returns the average of numerical values.
    170 
    171 * `COUNT(expr)`: returns the number of rows for which ***`expr`*** evaluates to a non-`NULL` value. `COUNT(*)` can be used to count all rows.
    172 
    173 * `MAX(expr)`: returns the maximum value, according to the datatype's ordering.
    174 
    175 * `MIN(expr)`: returns the minimum value, according to the datatype's ordering.
    176 
    177 * `SUM(expr)`: returns the sum of numerical values.

    179 ## SQL Statements
    180 
    181 ### `BEGIN`
    182 
    183 Starts a new [transaction](#transactions).
    184 
    185 <pre>
    186 BEGIN [ TRANSACTION ] [ READ ONLY | READ WRITE ] [ AS OF SYSTEM TIME <b><i>txn_id</i></b> ]
    187 </pre>
    188 
    189 * ***`txn_id`***: A past transaction ID to run a read-only transaction for, for time-travel queries.
    190 
    191 ### `COMMIT`
    192 
    193 Commits an active [transaction](#transactions).
    194 
    195 ### `CREATE TABLE`
    196 
    197 Creates a new table.
    198 
    199 <pre>
    200 CREATE TABLE <b><i>table_name</i></b> (
    201     [ <b><i>column_name</i></b> <b><i>data_type</i></b> [ <b><i>column_constraint</i></b> [ ... ] ]  [ INDEX ] [, ... ] ]
    202 )
    203 
    204 where <b><i>column_constraint</i></b> is:
    205 
    206 { NOT NULL | NULL | PRIMARY KEY | DEFAULT <b><i>expr</i></b> | REFERENCES <b><i>ref_table</i></b> | UNIQUE }
    207 </pre>
    208 
    209 * ***`table_name`***: The name of the table. Must be a [valid identifier](#identifiers). Errors if a table with this name already exists.
    210 
    211 * ***`column_name`***: The name of the column. Must be a [valid identifier](#identifiers), and unique within the table.
    212 
    213 * ***`data_type`***: The data type of the column, see [data types](#data-types) for valid types.
    214 
    215 * `NOT NULL`: The column may not contain `NULL` values.
    216 
    217 * `NULL`: The column may contain `NULL` values. This is the default.
    218 
    219 * `PRIMARY KEY`: The column should act as a primary key, i.e. the main row identifier. A table must have exactly one primary key column, and it must be unique and non-nullable.
    220 
    221 * `DEFAULT`***`expr`***: Specifies a default value for the column when `INSERT` statements do not give a value. ***`expr`*** can be any constant expression of an appropriate data type, e.g. `'abc'` or `1 + 2 * 3`. For nullable columns, the default value is `NULL` unless specified otherwise.
    222 
    223 * `REFERENCES`***`ref_table`***: The column is a foreign key to ***`ref_table`***'s primary key, enforcing referential integrity.
    224 
    225 * `UNIQUE`: The column may only contain unique (distinct) values. `NULL` values are not considered equal, thus a `UNIQUE` column which allows `NULL` may contain multiple `NULL` values. `PRIMARY KEY` columns are implicitly `UNIQUE`.
    226 
    227 * `INDEX`: Create an index for the column.
    228 
    229 #### Example
    230 
    231 ```sql
    232 CREATE TABLE movie (
    233     id INTEGER PRIMARY KEY,
    234     title STRING NOT NULL,
    235     release_year INTEGER INDEX,
    236     imdb_id STRING INDEX UNIQUE,
    237     bluray BOOLEAN NOT NULL DEFAULT TRUE
    238 )
    239 ```
    240 
    241 ### `DELETE`
    242 
    243 Deletes rows in a table.
    244 
    245 <pre>
    246 DELETE FROM <b><i>table_name</i></b>
    247     [ WHERE <b><i>predicate</i></b> ]
    248 </pre>
    249 
    250 Deletes rows where ***`predicate`*** evaluates to `TRUE`, or all rows if no `WHERE` clause is given.
    251 
    252 * ***`table_name`***: the table to delete from. Errors if it does not exist.
    253 
    254 * ***`predicate`***: an expression which determines which rows to delete by evaluting to `TRUE`. Must evaluate to a `BOOLEAN` or `NULL`, otherwise an error is returned.
    255 
    256 #### Example
    257 
    258 ```sql
    259 DELETE FROM movie
    260 WHERE release_year < 2000 AND bluray = FALSE
    261 ```
    262 
    263 ### `DROP TABLE`
    264 
    265 Deletes a table and all contained data. Errors if the table does not
    266 exist, unless `IF EXISTS` is given.
    267 
    268 <pre>
    269 DROP TABLE [ IF EXISTS ] <b><i>table_name</i></b>
    270 </pre>
    271 
    272 * ***`table_name`***: the table to delete.

    282 ### `INSERT`
    283 
    284 Inserts rows into a table.
    285 
    286 <pre>
    287 INSERT INTO <b><i>table_name</i></b>
    288     [ ( <b><i>column_name</i></b> [, ... ] ) ]
    289     VALUES ( <b><i>expression</i></b> [, ... ] ) [, ... ]
    290 </pre>
    291 
    292 If column names are given, an identical number of values must be given. If no column names are given, values must be given in the table's column order. Omitted columns will get a default value if specified, otherwise an error will be returned.
    293 
    294 * ***`table_name`***: the table to insert into. Errors if it does not exist.
    295 
    296 * ***`column_name`***: a column to insert into in the given table. Errors if it does not exist.
    297 
    298 * ***`expression`***: an expression to insert into the corresponding column. Must be a constant expression, i.e. it cannot refer to table columns.
    299 
    300 #### Example
    301 
    302 ```sql
    303 INSERT INTO movie
    304     (id, title, release_year)
    305 VALUES
    306     (1, 'Sicario', 2015),
    307     (2, 'Stalker', 1979),
    308     (3, 'Her', 2013)
    309 ```
    310 
    311 ### `ROLLBACK`
    312 
    313 Rolls back an active [transaction](#transactions).

    315 ### `SELECT`
    316 
    317 Selects rows from a table.
    318 
    319 <pre>
    320 SELECT [ * | <b><i>expression</i></b> [ [ AS ] <b><i>output_name</i></b> [, ...] ] ]
    321     [ FROM <b><i>from_item</i></b> [, ...] ]
    322     [ WHERE <b><i>predicate</i></b> ]
    323     [ GROUP BY <b><i>group_expr</i></b> [, ...] ]
    324     [ HAVING <b><i>having_expr</i></b> ]
    325     [ ORDER BY <b><i>order_expr</i></b> [ ASC | DESC ] [, ...] ]
    326     [ LIMIT <b><i>count</i></b> ]
    327     [ OFFSET <b><i>start</i></b> ]
    328 
    329 where <b><i>from_item</i></b> is one of:
    330 
    331 <b><i>table_name</i></b> [ [ AS ] <b><i>alias</i></b> ]
    332 <b><i>from_item</i></b> <b><i>join_type</i></b> <b><i>from_item</i></b> [ ON <b><i>join_predicate</i></b> ]
    333 
    334 where <b><i>join_type</i></b> is one of:
    335 
    336 CROSS JOIN
    337 [ INNER ] JOIN
    338 LEFT [ OUTER ] JOIN
    339 RIGHT [ OUTER ] JOIN
    340 
    341 </pre>
    342 
    343 Fetches rows or expressions, either from table ***`table_name`*** (if given) or generated.
    344 
    345 * ***`expression`***: [expression](#expressions) to fetch (can be a simple column name).
    346 
    347 * ***`output_name`***: output column [identifier](#identifier), defaults to column name (if single column) otherwise nothing (displayed as `?`).
    348 
    349 * ***`table_name`***: table to fetch rows from.
    350 
    351 * ***`alias`***: table alias.
    352 
    353 * ***`predicate`***: only return rows for which this [expression](#expressions) evaluates to `TRUE`.
    354 
    355 * ***`group_expr`***: an expression to group aggregates by. Non-aggregate `SELECT` expressions must either reference a column given in `group_expr`, be idential with a `group_expr`, or have an `output_name` that is referenced by a `group_expr` column.
    356 
    357 * ***`having_expr`***: only return aggregate results for which this [expression](#expressions) evaluates to `TRUE`.
    358 
    359 * ***`order_expr`***: order rows by this expression (can be a simple column name).
    360 
    361 * ***`count`***: maximum number of rows to return. Must be a constant integer expression.
    362 
    363 * ***`start`***: number of rows to skip. Must be a constant integer expression.
    364 
    365 * ***`join_predicate`***: only return rows for which this [expression](#expressions) evaluates to `TRUE`.
    366 
    367 Join types:
    368 
    369 * `CROSS JOIN`: returns the Carthesian product of the joined tables. Does not accept a join predicate (`ON` clause).
    370 
    371 * `INNER JOIN`: returns the rows of the tables' Carthesian product for which  ***`join_predicate`*** evaluates to `TRUE`.
    372 
    373 * `LEFT OUTER JOIN`: returns the rows joined on the ***`join_predicate`***, or for any rows in the left table that does not have a match in the right table a single row is returned with the right table's columns set to `NULL`.
    374 
    375 * `RIGHT OUTER JOIN`: the same as a `LEFT OUTER JOIN` but with the left and right tables switched.
    376 
    377 #### Example
    378 
    379 ```sql
    380 SELECT id, title, 2020 - released AS age
    381 FROM movies
    382 WHERE released >= 2000 AND ultrahd
    383 ORDER BY released DESC, title ASC
    384 LIMIT 10
    385 OFFSET 10
    386 ```
    387 
    388 ### `UPDATE`
    389 
    390 Updates rows in a table.
    391 
    392 <pre>
    393 UPDATE <b><i>table_name</i></b>
    394     SET <b><i>column_name</i></b> = <b><i>expression</i></b> | DEFAULT [, ... ]
    395     [ WHERE <b><i>predicate</i></b> ]
    396 </pre>
    397 
    398 Updates columns given by ***`column_name`*** to the corresponding ***`expression`*** for all rows where ***`predicate`*** evaluates to `TRUE`. If no `WHERE` clause is given, all rows are updated.
    399 
    400 * ***`table_name`***: the table to update. Errors if it does not exist.
    401 
    402 * ***`column_name`***: a column to update. Errors if it does not exist.
    403 
    404 * ***`expression`***: an expression whose evaluated value will be set for the corresponding column and row. Expressions can refer to column values, and must evaluate to the same datatype as the updated column. Using `DEFAULT` will set the column's default value, if any.
    405 
    406 * ***`predicate`***: an expression which determines which rows to update by evaluting to `TRUE`. Must evaluate to a `BOOLEAN` or `NULL`, otherwise an error is returned.
    407 
    408 #### Example
    409 
    410 ```sql
    411 UPDATE movie
    412 SET bluray = TRUE
    413 WHERE release_year >= 2000 AND bluray = FALSE
    414 ```

    416 ## Transactions
    417 
    418 toyDB supports ACID transactions using MVCC-based snapshot isolation, protecting from the following anomalies: dirty writes, dirty reads, lost updates, fuzzy reads, read skew, and phantom reads. However, write skew anomalies are possible since serializable snapshot isolation is not implemented.
    419 
    420 A new transaction is started with `BEGIN`, and ended with either `COMMIT` (atomically writing all changes) or `ROLLBACK` (discarding all changes). If any conflicts occur between concurrent transactions, the lowest transaction ID wins and the others will fail with a serialization error and must retry.
    421 
    422 All past data is versioned and retained, and can be queried as of a given transaction ID via `BEGIN TRANSACTION READ ONLY AS OF SYSTEM TIME <txn_id>`.
    423 
    424 A transaction is still valid for use if a contained statement returns an error. It is up to the client to take appropriate action.
